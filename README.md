ARP parser realization with C and Erlang. Build instructions provided in each directory.


## Тестовое задание: ARP-парсер
Цель: Написать парсер ARP-пакета на языках Erlang и C.
Технические требования:
1.	Формат ARP-пакета:
•	Hardware type (2 байта)
•	Protocol type (2 байта)
•	Hardware size (1 байт)
•	Protocol size (1 байт)
•	Opcode (2 байта)
•	Sender MAC (6 байт)
•	Sender IP (4 байта)
•	Target MAC (6 байт)
•	Target IP (4 байта)

2.	Реализация на Erlang:
•	Использовать record для хранения состояния
•	Функция должна принимать бинарные данные и возвращать заполненный record
•	Обработка ошибок (некорректный размер пакета и т.д.)

3.	Реализация на C:
•	Использовать структуру для хранения данных
•	Функция должна принимать указатель на сырые данные и заполнять структуру
•	Проверка корректности данных

Критерии оценки:
1.	Корректность разбора всех полей
2.	Обработка ошибок (недостаточная длина пакета и т.д.)PSize
3.	Читаемость кода

Пример ARP-пакета (28 байт):

00 01 08 00 06 04 00 01
08 00 27 12 34 56 C0 A8
01 01 00 00 00 00 00 00
C0 A8 01 02
